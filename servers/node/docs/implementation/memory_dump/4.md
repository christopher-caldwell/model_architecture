## Architectural Decisions

All multi-aggregate business workflows are in command classes. Transports will call these methods and should not call domain transition functions directly.

`inWriteUnitOfWork` centralizes transaction lifecycle:

- build UoW
- run command body
- commit on success
- rollback on failure
- preserve original failure if rollback fails

Domain errors are thrown directly so error identity is preserved. `UnexpectedCommandError` exists for future infrastructure wrapping if needed, but command methods do not flatten domain errors into transport-style categories.

Queries use read repositories directly and do not touch the write unit of work.
