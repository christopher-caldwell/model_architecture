## Architectural Context

The Node / TypeScript server is intended as a faithful interpretation of the Rust server, not a direct translation.

Core rules captured in the plan:

- Business logic belongs only in domain behavior and application command/query handlers.
- Transports remain thin: parse request shape, map DTOs, call application, map errors.
- Commands own mutating workflows and use a transactional write unit of work.
- Queries own read workflows and use read repositories directly.
- HTTP DTOs and GraphQL DTOs are separate transport contracts.
- Import boundaries mirror Rust crate dependency direction.

Target packages:

- `@library/domain`
- `@library/application`
- `@library/auth-core`
- `@library/persistence`
- `@library/server-bootstrap`
- `@library/http-server`
- `@library/graphql-server`

## Checkpoint Protocol

The protocol is now captured in `todo.json`:

- Before starting a task, set status to `in_progress`.
- When complete, set status to `done`.
- After each completed task, write a memory dump under `memory_dump/`.
- Each memory dump must capture changes, files touched, decisions, verification, and next-task context.
- After each memory dump, compact context to what is needed to continue.
