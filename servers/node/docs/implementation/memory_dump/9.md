## GraphQL Design

GraphQL transport has its own contract types:

- `CatalogTitle`
- `InventoryCopy`
- `LibraryMember`
- `LoanRecord`
- GraphQL input types in SDL:
  - `CreateCatalogTitleInput`
  - `AddInventoryCopyInput`
  - `RegisterMemberInput`
  - `StartLoanInput`

Resolvers are thin:

- read args
- call `ServerDeps` command/query services
- map results into GraphQL object shapes
- map errors into GraphQL errors

The package does not import domain/application directly. It imports only:

- `@library/server-bootstrap`
- `@library/auth-core`

## Auth And Errors

GraphQL POST `/graphql` is protected with bearer auth before Apollo middleware runs. GET `/graphql` returns a simple public text response.

Error mapping uses error `name` strings to keep import boundaries strict:

- not found -> GraphQL extension code `NOT_FOUND`
- business conflicts -> `CONFLICT`
- unknown errors -> `INTERNAL_SERVER_ERROR`
