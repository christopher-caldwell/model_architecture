## Documentation Context

Checked official Pothos docs:

- Overview: Pothos is a plugin-based TypeScript GraphQL schema builder, with `graphql` as core dependency.
- Guide: create a `SchemaBuilder`, define object/query fields, and call `builder.toSchema()`.
- Objects: `builder.objectRef<T>()` defines typed object refs that are implemented with explicit fields.
- Queries/mutations: `builder.queryType()` and `builder.mutationType()` define root fields.
- Enums: `builder.enumType()` supports string arrays and values objects.

## Architecture Notes

The GraphQL package still imports only:

- `@library/server-bootstrap`
- `@library/auth-core`

It does not import domain or application directly.

Resolvers remain thin:

- read GraphQL args
- call `ctx.deps.*.commands` or `ctx.deps.*.queries`
- map application results to GraphQL object contract types
- map errors to GraphQL error extensions

GraphQL object/input contracts remain separate from HTTP DTOs and domain/application types.
