## HTTP Design

Transport is intentionally thin:

- parse params/body
- map HTTP request DTOs into application command inputs
- call `ServerDeps` command/query services
- map application results into HTTP response DTOs
- map errors into HTTP responses

## Auth And Errors

`authMiddleware` reads `Authorization: Bearer <token>`, calls `deps.auth.verifier.verifyToken`, and stores claims on the request object.

HTTP transport does not import `@library/domain` or `@library/application`. Domain/application errors are mapped by stable error `name` strings:

- not found -> `404`
- business conflicts -> `409`
- unknown errors -> `500`

This keeps import boundaries strict while preserving transport-owned response semantics.

## TypeScript Note

The `asyncHandler` wrapper accepts broad `req`/`res` types so individual route callbacks can keep route-specific `Request<{ params }, ..., Body>` typing. This avoids Express generic variance friction without weakening architectural boundaries.
